# AWS CLI S3 Cheatsheet (RunPod)

> Assumes `~/.aws/config` has `endpoint_url` and you're using `--profile runpod`.
> Replace `s3://my-bucket` with your actual bucket name.

---

## Delete

```bash
# Delete a single file
aws s3 rm s3://my-bucket/path/to/file.txt --profile runpod

# Delete a directory and all its contents (recursive)
aws s3 rm s3://my-bucket/path/to/folder/ --recursive --profile runpod

# Delete all files matching a pattern
aws s3 rm s3://my-bucket/ --recursive --exclude "*" --include "*.log" --profile runpod
```

---

## Upload

```bash
# Upload a single file
aws s3 cp myfile.txt s3://my-bucket/path/to/myfile.txt --profile runpod

# Upload a single file into a folder (keep filename)
aws s3 cp myfile.txt s3://my-bucket/some-folder/ --profile runpod

# Upload a directory with all files and subdirectories
aws s3 cp ./my-folder s3://my-bucket/my-folder --recursive --profile runpod

# Sync a local directory (only uploads new/changed files, skips unchanged)
aws s3 sync ./my-folder s3://my-bucket/my-folder --profile runpod

# Sync and delete files on remote that no longer exist locally
aws s3 sync ./my-folder s3://my-bucket/my-folder --delete --profile runpod
```

---

## Download

```bash
# Download a single file
aws s3 cp s3://my-bucket/path/to/file.txt ./file.txt --profile runpod

# Download a directory with all files and subdirectories
aws s3 cp s3://my-bucket/my-folder ./ --recursive --profile runpod

# Sync remote directory to local (only downloads new/changed files)
aws s3 sync s3://my-bucket/my-folder ./my-folder --profile runpod
```

---

## List

```bash
# List all buckets
aws s3 ls --profile runpod

# List files in a bucket (top level)
aws s3 ls s3://my-bucket/ --profile runpod

# List files in a subfolder
aws s3 ls s3://my-bucket/path/to/folder/ --profile runpod

# List all files recursively (with sizes)
aws s3 ls s3://my-bucket/ --recursive --human-readable --profile runpod

# List and summarize total size
aws s3 ls s3://my-bucket/ --recursive --human-readable --summarize --profile runpod
```

---

## Move / Rename

```bash
# Move (rename) a file on remote
aws s3 mv s3://my-bucket/old-name.txt s3://my-bucket/new-name.txt --profile runpod

# Move a file from local to remote (upload + delete local)
aws s3 mv myfile.txt s3://my-bucket/myfile.txt --profile runpod

# Move a directory recursively on remote
aws s3 mv s3://my-bucket/old-folder/ s3://my-bucket/new-folder/ --recursive --profile runpod
```

---

## Useful Flags

| Flag | Description |
|------|-------------|
| `--recursive` | Apply operation to all files in directory |
| `--delete` | (sync only) Remove remote files not in local source |
| `--dryrun` | Preview what would happen without executing |
| `--exclude "pattern"` | Skip files matching pattern |
| `--include "pattern"` | Include files matching pattern (use with `--exclude "*"`) |
| `--human-readable` | Show sizes in KB/MB/GB |
| `--summarize` | Show total count and size at end of `ls` |
| `--no-progress` | Hide progress bar (cleaner for scripts/logs) |
| `--only-show-errors` | Suppress all output except errors |

---

## Tips

```bash
# Preview a destructive sync before running it
aws s3 sync ./my-folder s3://my-bucket/my-folder --delete --dryrun --profile runpod

# Exclude a subdirectory from upload
aws s3 cp ./my-folder s3://my-bucket/my-folder --recursive \
  --exclude "node_modules/*" --profile runpod

# Set AWS_PROFILE to avoid typing --profile every time
export AWS_PROFILE=runpod
```
